DOM Basics

  • "Document Object Model"
  • Standard maintained by the W3C (the same nerds who brought you the HTML and CSS specs)
  • A way to interact with the HTML elements on a webpage
  • Uses the standard document object in JavaScript
  • Provides a structured representation of HTML elements

DOM Structure

The DOM is made up of nodes that are connected (and named) like a family tree.

The first node is known as the root.

In HTML, the root is always the <html> element

Nodes are often referred to based on their relationships with other nodes.

The common relationships are parent, child, and sibling.

DOM Structure

<head> and <body> are siblings.

<head> is the parent of <title>.

<h1> is the child of <body>.

DOM Interaction

Access nodes via their relationship with other nodes.


  var bodyNode = document.body; // <body>
  var htmlNode = document.body.parentNode; // <html>
  for (var i = 0; i < document.body.childNodes.length; i++) {
    var childNode = document.body.childNodes[i];
    //could be <p>, <h1>, etc.
    //any html element
  }

DOM Interaction: Easier

Finding every element on the page by siblings and children is time consuming!

The document object provides methods for finding DOM nodes without going one by one

DOM Interaction: Finding Elements

getElementById - gets an element by its ID attribute; method of the document object. 
<img id="mainpicture" src="http://girldevelopit.com/assets/pink-logo.png">

var img = document.getElementById('mainpicture');
getElementsByTagName - gets a set of elements by their HTML tag name (such as div, p, etc.); method of the document object OR a node. 
<li class="peanut">Charlie Brown</li>
<li class="peanut">Linus van Pelt</li>

var listItems = document.getElementsByTagName('li');
for (var i = 0; i < listItems.length; i++) {
   var listItem = listItems[i];
}

DOM Interaction: Finding Elements

getElementsByClassName - gets a set of elements by their class attribute; method of the document object OR a node. 
<img class="image-large" src="http://www.girldevelopit.com/assets/pink-logo.png" />
<img class="image-large" src="http://www.girldevelopit.com/assets/community.png" />

var largeImages = document.getElementsByClassName('image-large');
for (var i = 0; i < largeImages.length; i++) {
   var image = largeImages[i];
}

DOM Modifying -- Creating

The document object can create new nodes, modify node attributes, and modify node content.

There are two methods for creating nodes.

document.createElement - creates a new HTML element

var paragraph = document.createElement('p');

document.createTextNode - creates a new text node

var paragraphText = document.createTextNode('New Paragraph!');

DOM Modifying -- Adding and Removing

parentNode.appendChild(childNode) - appends childNode as the last child of parentNode.

var paragraph = document.createElement('p');
var paragraphText = document.createTextNode('New Paragraph!');
paragraph.appendChild(paragraphText);
document.body.appendChild(paragraph);

parentNode.removeChild(childNode) - removes the specified childNode.

document.body.removeChild(paragraphText);

DOM Nodes -- Attributes

We can use node methods to set and retrieve attributes

getAttribute - retrieves an HTML attribute of a node 
var img = document.getElementById('mainpicture');
var imgUrl = img.getAttribute('src');
document.write(imgUrl);
setAttribute - sets an HTML attribute of a node 
var img = document.getElementById('mainpicture');
img.setAttribute('src', 'new-picture.jpeg');

DOM innerHTML

Each DOM node has an innerHTML property:

document.body.innerHTML;
You can set innerHTML yourself to change the contents of the node: 
document.body.innerHTML = '<p>I changed the whole page!</p>';
You can also just add to the innerHTML instead of replace everything: 

document.body.innerHTML += "...just adding this bit at the end of the page.";

Let's Answer It (Favorite Things)

<div id="favorite-things"></div>

function favoriteThings(){
    var favoriteThings = ['Rabbits', 'Orange', 'Yogurt', 'Brussel Sprouts', 'Otters'];
    var resultDiv = document.getElementById('favorite-things');
    
    var resultParagraph = document.createElement('p');
    var result = 'My favorite things are: ';
    var resultText;
    
    for (var i = 0; i < favoriteThings.length; i++) {
        if (i < favoriteThings.length - 1) {
            result += favoriteThings[i] + ', ';
        }
        else {
            result += 'and ' + favoriteThings[i] + '.';
        }
    }
    resultText = document.createTextNode(result);
    resultParagraph.appendChild(resultText);
    resultDiv.appendChild(resultParagraph);
}

DOM Events -- Introduction

Events are at the core of Web applications.

Events allow a function (known as an event handler) to run when an action is performed.

We've already implemented handlers--the functions for onclick on our buttons!

Hooking Up Events

Attaching/Detaching - fancy terminology for telling the DOM to add or remove an event handler.

Three ways of attaching events to a node.

  • on* attributes (in HTML like we've done already)
  • on* properties
  • addEventListener (or attachEvent) method

<div id="change-me">Hello, world!</div>
function changeText() {
   this.innerHTML = 'My click event changed me!';
}

var changeDiv = document.getElementById('change-me');
changeDiv.onclick = changeText;
changeDiv.addEventListener('click', changeText);
Hello, world!

Hooking Up Events

addEventListener vs. on* properties: What's the difference?

  • on* properties overwrite all existing handlers.
  • addEventListener adds to the list of handlers.
function changeColor(event) {
   this.style.backgroundColor = '#999';
}
changeDiv.onclick = changeText;
changeDiv.onclick = changeColor;
Hello, world!
changeDiv.addEventListener('click', changeText);
changeDiv.addEventListener('click', changeColor);
Hello, world!

Mouse Events

  • onclick/click - event that fires when a user clicks on an element
  • onmouseover/mouseover - event that fires when a user moves the cursor over an element
  • onmouseout/mouseout - event that fires when the cursor exits an element
  • onmousedown/mousedown - event that fires when the mouse button is pressed down on an element
  • onmouseup/mouseup - event that fires when the mouse button is released on an element
Hello, world!

Other Important Events

  • onkeydown/keydown - fires when a key has been pressed.
    0
  • onkeyup/keyup - fires when a key has been released.
    0
  • onfocus/focus - fires when an element receives focus
  • onblur/blur - fires when an element loses focus.
  • DOMContentLoaded - fires when the page has been parsed but before resources (images, for example) have downloaded. This can *only* be used with addEventListener.
  • And many, many more
